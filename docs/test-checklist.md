# Test checklist

## User Service

- [x] Регистрация с корректными данными возвращает 201/200.
- [x] Повторная регистрация с тем же email возвращает 409.
- [x] Регистрация с пустым email возвращает 400.
- [x] Регистрация с пустым password возвращает 400.
- [x] Логин с правильными данными возвращает token/user.
- [x] Логин с неверным паролем возвращает 401.
- [x] `GET /users/me` без авторизации возвращает 401.
- [x] `GET /users/me` с авторизацией возвращает профиль.
- [x] `PUT /users/me` меняет данные профиля.
- [x] `POST /users/me/addresses` добавляет адрес.
- [x] `GET /users/me/addresses` возвращает список адресов.

## Catalog Service

Status: blocked, project is not present in the repository yet.

- [ ] `GET /categories` возвращает список категорий.
- [ ] `GET /dishes` возвращает список блюд.
- [ ] В базе есть минимум 8-12 блюд.
- [ ] `GET /dishes?categoryId=1` фильтрует блюда.
- [ ] `GET /dishes?search=чак` ищет блюда.
- [ ] `GET /dishes/{id}` возвращает карточку блюда.
- [ ] Несуществующее блюдо возвращает 404.
- [ ] У блюда есть поле доступности, например `isAvailable`.

## Order Service

Status: blocked, project is not present in the repository yet.

- [ ] `POST /orders` создает заказ с валидными данными.
- [ ] Заказ содержит позиции заказа.
- [ ] Сумма заказа считается корректно.
- [ ] `GET /orders/{id}` возвращает заказ.
- [ ] `GET /orders/my` возвращает заказы пользователя.
- [ ] `POST /orders/{id}/cancel` отменяет заказ.
- [ ] Нельзя отменить уже оплаченный/доставленный заказ, если такое правило есть.
- [ ] Заказ с пустым `items` возвращает 400.
- [ ] Заказ с несуществующим `dishId` возвращает 400 или 404.
