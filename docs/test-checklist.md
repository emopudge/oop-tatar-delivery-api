# Test checklist

## User Service

- [x] Регистрация с корректными данными возвращает 201/200.
- [x] Повторная регистрация с тем же email возвращает 409.
- [x] Регистрация с пустым email возвращает 400.
- [x] Регистрация с пустым password возвращает 400.
- [x] Логин с правильными данными возвращает token/user.
- [x] Логин с неверным паролем возвращает 401.
- [x] `GET /users/me` без авторизации возвращает 401.
- [x] `GET /users/me` с авторизацией возвращает профиль.
- [x] `PUT /users/me` меняет данные профиля.
- [x] `POST /users/me/addresses` добавляет адрес.
- [x] `GET /users/me/addresses` возвращает список адресов.

## Catalog Service

- [x] `GET /categories` возвращает список категорий.
- [x] `GET /dishes` возвращает список блюд.
- [x] В базе есть минимум 8-12 блюд.
- [x] `GET /dishes?categoryId=1` фильтрует блюда.
- [x] `GET /dishes?search=%D1%87%D0%B0%D0%BA` ищет блюда.
- [x] `GET /dishes/{id}` возвращает карточку блюда.
- [x] Несуществующее блюдо возвращает 404.
- [x] У блюда есть поле доступности, например `isAvailable`.

## Order Service

- [x] `POST /orders` создает заказ с валидными данными.
- [x] Заказ содержит позиции заказа.
- [x] Сумма заказа считается корректно по mock-ценам Order Service.
- [x] `GET /orders/{id}` возвращает заказ.
- [x] `GET /orders/my?userId=1` возвращает заказы пользователя.
- [x] `POST /orders/{id}/cancel` отменяет заказ в статусе `PendingPayment`.
- [x] Нельзя отменить уже оплаченный/доставленный заказ, если такое правило есть.
- [x] `POST /orders/{id}/pay` переводит заказ в `Paid`.
- [x] `POST /orders/{id}/deliver` переводит оплаченный заказ в `Delivered`.
- [x] Заказ с пустым `items` возвращает 400.
- [ ] Заказ с несуществующим `dishId` возвращает 400 или 404.
